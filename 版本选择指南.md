## 🎯 选择建议

### 选择 V1 (基础版) 如果您需要:

✅ **永久授权模式**
- 用户一次付费，终身使用
- 不需要订阅或续费
- 一人一码，但无时间限制

✅ **简单实用**
- 不需要管理有效期
- 不需要提醒用户续费
- 密钥一旦分配即可长期使用

✅ **适用场景**
- 买断制商业软件
- 内部工具永久授权
- VIP用户终身权限
- 不想处理续费问题

**示例场景:**
```
场景1: 企业内部管理系统
- 员工一旦获得权限，长期有效
- 除非离职，否则不需要重新授权

场景2: 专业工具软件
- 用户付费购买后永久使用
- 买断制，无需订阅

场景3: 付费插件/扩展
- 一次付费，终身授权
- 无需管理有效期
```

---

### 选择 V2 (时效性版) 如果您需要:

✅ **订阅制/会员制**
- 月卡、季卡、年卡模式
- 需要用户定期续费
- 试用期控制

✅ **灵活的授权策略**
- 不同用户不同有效期
- 支持试用转正式
- 临时授权自动过期

✅ **收益最大化**
- 持续性收入
- 防止一次付费永久使用
- 用户分级管理

✅ **适用场景**
- SaaS订阅软件
- 在线教育课程
- 会员制服务
- 游戏月卡系统
- 临时项目授权

**示例场景:**
```
场景1: 在线教育平台
- 学员购买30天/90天/365天课程
- 到期后需要续费才能继续学习

场景2: 设计工具软件
- 基础版: ¥49/月
- 专业版: ¥399/年
- 试用版: 7天免费

场景3: 企业临时项目

- 实习生: 90天试用期
```

## 📋 详细功能对比

### V1 (基础版)

**密钥生成:**
```bash
python tools/generate_keys.py
```

**密钥特点:**
- 64位十六进制字符串
- 每个用户独立密钥
- 密钥永久有效

**用户体验:**
```
1. 输入密钥
2. 验证成功 → 程序启动
3. 以后每次启动都成功（只要密钥正确）
```

**管理功能:**
- 生成密钥
- 查看密钥列表
- 导出密钥清单
- 批量生成

**文件:**
- `tools/encrypt_exe.py`
- `tools/generate_keys.py`
- `dist/launcher.py`

---

### V2 (时效性版)

**密钥生成:**
```bash
python tools/generate_keys_v2.py
```

**密钥特点:**
- 64位十六进制字符串
- 每个用户独立密钥
- 包含有效期信息
- 支持1天-永久各种期限

**用户体验:**
```
首次使用:
1. 输入密钥
2. "首次激活成功！有效期 30 天"
3. 程序启动

日常使用:
1. 自动读取激活记录
2. "密钥有效，还剩 25 天"
3. 程序启动

有效期不足:
1. "密钥有效，还剩 5 天"
2. 弹窗提醒: "请及时续期！"
3. 程序启动

密钥过期:
1. "密钥已过期"
2. 弹窗提示: "请联系管理员续期"
3. 程序拒绝启动
```

**管理功能:**
- 生成密钥（含有效期）
- 查看密钥状态（剩余天数）
- 延长有效期
- 导出密钥清单
- 批量生成
- 过期检测

**文件:**
- `tools/encrypt_exe.py` (共用)
- `tools/generate_keys_v2.py`
- `dist/launcher_v2.py`

## 🔄 版本切换和混用

### 可以同时使用吗？

✅ **可以！两个版本完全独立**

- V1和V2可以同时存在
- 使用不同的文件名
- 互不干扰

### 混合使用场景

```
企业应用场景:
├── 正式员工 → 使用V2 (365天年卡)
├── 实习生 → 使用V2 (90天试用)
├── 核心团队 → 使用V1 (永久授权)
└── 临时外包 → 使用V2 (30天项目期)
```

```
商业软件场景:
├── 普通用户 → 使用V2 (月卡/年卡订阅)
├── VIP用户 → 使用V1 (永久授权)
└── 试用用户 → 使用V2 (7天试用)
```

## 📈 迁移建议

### 从V1迁移到V2

**适用情况:**
- 想要实现订阅制
- 需要控制使用时长
- 希望增加持续性收入

**迁移步骤:**
1. 保留V1工具和文件
2. 安装V2版本
3. 新用户使用V2密钥
4. 老用户继续使用V1密钥
5. 老用户密钥到期后提供V2升级

**注意事项:**
- 提前通知用户模式变更
- 给老用户优惠价格
- 提供平滑过渡期

---

### 从V2切换到V1

**适用情况:**
- 用户反馈不喜欢订阅制
- 管理有效期太麻烦
- 决定采用买断制

**切换步骤:**
1. 安装V1版本
2. 重新生成永久密钥
3. 通知用户更换密钥
4. 逐步淘汰V2

## 🎯 推荐选择

### 我应该选择哪个版本？

**选择 V1 如果:**
- ✅ 您的软件适合买断制
- ✅ 用户期望一次付费永久使用

- ✅ SaaS模式或会员制服务
- ✅ 临时授权需求

**不确定？**
- 👉 从V2开始，保留V1选项
- 👉 提供多种授权方式
- 👉 根据用户反馈调整
